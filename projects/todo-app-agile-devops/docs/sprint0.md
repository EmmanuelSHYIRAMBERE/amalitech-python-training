Sprint 0 — Planning

Product Vision
Build a simple web-based To-Do application that allows users to create, manage, and track tasks efficiently.

Product Backlog

User Story 1: Create Task
As a user, I want to create a task so I can track my work.

Acceptance Criteria:

- User can enter task title
- Task is saved
- Confirmation is shown

Priority: High
Story Points: 3

User Story 2: View Tasks
As a user, I want to see all tasks.

Acceptance Criteria:

- All tasks are displayed
- Each task shows title and status

Priority: High
Story Points: 2

User Story 3: Mark Task Complete
As a user, I want to mark a task as complete.

Acceptance Criteria:

- User can update task status
- Status is reflected immediately

Priority: High
Story Points: 3

User Story 4: Delete Task
As a user, I want to delete a task.

Acceptance Criteria:

- Task can be removed
- UI updates accordingly

Priority: Medium
Story Points: 2

User Story 5: Task Summary
As a user, I want to see completed vs pending tasks.

Acceptance Criteria:

- System shows counts of completed and pending tasks

Priority: Medium
Story Points: 2

Definition of Done (DoD)

- Feature implemented
- Code committed to GitHub
- Tests written and passing
- CI pipeline passes
- Meets acceptance criteria
