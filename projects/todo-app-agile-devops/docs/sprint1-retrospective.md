Sprint 1 Retrospective — To-Do Web Application

What Went Well

- Successfully implemented core task management features
- CI/CD pipeline was set up and is functioning correctly
- Maintained incremental commits instead of large final commits
- Integrated testing into the development workflow

What Didn’t Go Well

- Initial CI pipeline failed due to test detection issues
- Minor difficulties with project structure and imports
- Time spent debugging environment configuration

Lessons Learned

- Tests must follow proper naming conventions for CI detection
- Running tests locally before pushing avoids pipeline failures
- Structuring the project early helps reduce refactoring later

Improvements for Sprint 2

1. Improve Development Workflow

- Run tests locally before every push
- Use smaller and more frequent commits

2. Enhance Application Features

- Add task deletion functionality
- Implement task summaries

3. Improve Code Quality

- Add more comprehensive tests
- Refactor code for better organization

Action Plan for Sprint 2

- Implement delete task feature
- Add summary endpoint (completed vs pending)
- Introduce logging for monitoring
- Expand test coverage including edge cases
