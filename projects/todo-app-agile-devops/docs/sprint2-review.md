Sprint 2 Review — To-Do Web Application

Overview

Sprint 2 focused on improving the application by adding new features, enhancing code quality, and introducing monitoring through logging.

Features Delivered

1. Task Deletion
   Users can now delete tasks via a DELETE endpoint.

2. Task Summary
   A summary endpoint was added to display the number of completed and pending tasks.

3. Logging & Monitoring
   Logging was implemented to track application events such as task creation, completion, and deletion.

4. Improved Testing
   Additional tests were added to improve coverage and ensure application reliability.

Improvements from Sprint 1

- Better test coverage
- More structured and modular code
- Increased number of commits
- Improved application observability with logging

Conclusion

Sprint 2 successfully enhanced the To-Do application by adding new features and improving system reliability. The application now demonstrates both Agile improvements and DevOps practices.
